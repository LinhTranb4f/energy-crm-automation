| HubSpot Quote→Deal sync ignores quotes without discounts | Issue | High | In the "HubSpot | Update deal Amount to match new quote amount" workflow, if a quote has no associated discounts the `If2` node’s FALSE branch is unconnected, so execution stops and the deal amount is never updated. This can leave deals with amounts that no longer match their latest quote totals. | Connect the FALSE branch of `If2` to a calculation path that handles the "no discounts" scenario by summing the quote’s line items directly and updating the deal amount accordingly. | Hardcoded tokens in quote→deal amount workflow | Improvement | High | The "HubSpot | Update deal Amount to match new quote amount" workflow embeds hardcoded Bearer tokens in multiple places (Global variables code node, JavaScript code node, and the "Update amount deal" HTTP request headers), which is explicitly called out as a critical security risk. | Move all tokens into n8n credential objects or environment variables and reference them via n8n’s credentials system instead of hardcoding them. It also lacks explicit error handling, so MongoDB or n8n API failures will surface as unhandled errors and the webhook caller simply times out. | Replace the destructive delete+insert pattern with a safer two-phase update (e.g., write to a temporary collection and swap, or use upserts keyed by workflow ID). Wrap MongoDB and n8n API calls in explicit error handling, ensure the webhook returns a clear error payload on failure, and consider adding alerting when the sync fails so the workflows directory is never silently empty. | [Sync Workflows to MongoDB](https://app.clickup.com/24322455/v/dc/q68cq-250875/q68cq-621955) |  |  |
