This workflow serves as the central configuration utility for the broader automation ecosystem. # Security Notes

*   Hardcoded System Constants: The workflow stores environment-specific configuration values (such as HubSpot Portal ID, Pipeline IDs, and Owner IDs) as hardcoded JavaScript variables. While these are not highly sensitive credentials (like API keys or OAuth tokens), they map out the architecture of the connected HubSpot environment and should be protected from unauthorized modification.

# Known Limitations / Risks

*   Hardcoded Logic Dependency: Because all variables are explicitly hardcoded within the code node, any structural change in HubSpot (such as deleting and recreating a pipeline, or editing a custom association label) will immediately break downstream workflows until this sub-workflow is manually updated.
*   Static Return: There is no dynamic validation to check if the returned HubSpot IDs are still valid or active within the external CRM.

# Final Outputs

The workflow seamlessly produces a robust JSON payload containing the complete set of global variables, HubSpot IDs, and association definitions. This output is handed directly back to the invoking parent workflow to drive external API calls and business logic execution.
